# Денис Коваленко — фотограф

Статический сайт-визитка для фотографа Дениса Коваленко. Проект сделан без backend и без внешних зависимостей: только HTML, CSS и JavaScript. Его можно сразу публиковать на GitHub Pages.

## Как запустить локально

Откройте `index.html` в браузере или запустите простой локальный сервер:

```bash
python3 -m http.server 8080
```

После этого сайт будет доступен по адресу `http://localhost:8080`.

## Как собрать проект

Сборка не нужна. Все рабочие файлы уже лежат в корне проекта:

- `index.html` — структура, SEO и Open Graph;
- `styles/main.css` — визуальная система, адаптив и анимации;
- `scripts/main.js` — квиз, умный отчет заявки, календарь съемочных окон, копирование текста, scroll reveal и пасхалка;
- `assets/favicon/favicon.svg` — favicon;
- `assets/svg/og-cover.svg` — честная типографическая OG-обложка без фейковых фото.

## Деплой на GitHub Pages

1. Загрузите проект в GitHub-репозиторий.
2. В настройках репозитория откройте `Settings` → `Pages`.
3. В `Build and deployment` выберите `Deploy from a branch`.
4. Укажите ветку `main` и папку `/root`.
5. Сохраните настройки.

Так как проект использует относительные пути, он корректно работает и в подпапке GitHub Pages.

Основные тексты находятся в `index.html`. Там же редактируются гайды, пакеты, чеклист подготовки и блок факторов стоимости. Вопросы, варианты ответов и финальный отчет заявки редактируются в `scripts/main.js` в массиве `quizSteps` и функции `buildMessage`.

## Где добавлять фотографии

Создайте папку, например `assets/photos/`, и положите туда реальные фотографии Дениса. После этого замените нужные `figure` с классами `frame` или `portfolio-frame` на изображения:

```html
<figure class="portfolio-frame">
  <img src="assets/photos/portrait-01.jpg" alt="Портретная съемка Дениса Коваленко" />
  <figcaption>portrait / 2026</figcaption>
</figure>
```

В CSS для таких изображений можно добавить:

```css
.portfolio-frame img,
.frame img {
  width: 100%;
  height: 100%;
  object-fit: cover;
}
```

## Как устроен квиз

Квиз работает полностью на клиенте:

1. Пользователь выбирает ответы.
2. На шаге с датой появляется дополнительное поле, если выбран вариант `Дата уже есть`.
3. На шаге `Что важно?` можно выбрать несколько вариантов.
4. В финале JS собирает структурированный отчет с эмодзи.
5. Отчет дополнительно показывает, что уже понятно, предварительный формат, факторы стоимости и вопросы, которые осталось уточнить.
6. Пользователь копирует текст и открывает Telegram или VK.

Backend, формы и отправка на сервер не используются.

## Как устроен календарь

Календарь съемочных окон считается в `scripts/main.js` в объекте `schedule`. Сейчас старт цикла — `6 июля 2026`: первые 15 дней отмечены как съемочные, следующие 15 дней — как даты по запросу. Чтобы изменить график, поменяйте `start`, `shootingDays` или `requestDays`.

## Как заменить плейсхолдеры на реальные фото

Плейсхолдеры намеренно сделаны графическими: линии, рамки, подписи и пустые кадры. Когда появятся реальные работы, заменяйте только содержимое кадров, сохраняя подписи и сетку. Не добавляйте stock-фото или AI-generated изображения: сайт рассчитан на честное портфолио реального фотографа.
